# 2-define_dialect

## Purpose
Chapter 2: Define the minimal Siren dialect skeleton. Introduces the core MLIR pattern of declaring a dialect via TableGen (`.td`) and wiring it into an `mlir-opt`-style tool. No ops are defined yet — just the dialect registration boilerplate that every subsequent chapter builds upon.

| File | Description |

| `CMakeLists.txt` | Top-level CMake for this chapter; delegates to subdirs |

| `include/npu-mlir/Dialect/Siren/IR/SirenDialect.td` | TableGen dialect definition — `def SirenDialect` |

## For AI Agents

### Working In This Directory
- This is the base pattern for all later chapters; keep changes minimal.
- To add ops, interfaces, or traits, work in `3-define_op/` and above, not here.
- The `SirenDialect.td` file defines namespace `npu_mlir` and dialect name `"siren"`.

### Testing Requirements
```bash
TARGET=ch-2 ./build_npu_mlir_fixed.sh
install/bin/ch-2-opt --show-dialects   # should list 'siren'
```

### Common Patterns
- `SirenDialect.h` does `#include "SirenDialect.h.inc"` — the `.inc` is TableGen-generated.
- `SirenDialect.cpp` calls `initialize()` generated by TableGen.
- `npu-opt.cpp`: `registry.insert<mlir::npu_mlir::SirenDialect>()` is the registration idiom.
